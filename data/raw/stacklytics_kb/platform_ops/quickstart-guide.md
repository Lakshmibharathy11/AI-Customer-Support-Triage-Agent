# Quickstart guide

## Introduction to Stacklytics
Stacklytics is a B2B product-analytics SaaS designed for engineering teams, offering three customer tiers: Gold, Diamond, and Platinum. The Gold tier is the entry tier, which includes 1 seat, 1M events/mo, and standard support response in 8 hours. The Diamond tier is the mid tier, offering up to 10 seats, 10M events/mo, and priority support response in 4 hours. The Platinum tier is the top tier, providing unlimited seats, unlimited events, SSO/SAML, a dedicated CSM, and support response in 1 hour.

## Getting Started with the API
To start using Stacklytics, you can access our API at the base URL https://api.stacklytics.io/v2. We also offer SDKs in JavaScript, Python, and Go to help you integrate our service with your application. When using the API, be aware of common errors such as 401 (invalid API key) and 429 (rate limit exceeded, error code RATE_LIMIT_EXCEEDED), as well as SDK-1042 (event schema validation failure). Make sure to check the API documentation for more information on how to handle these errors.

## Support and Resources
Our support team is available Monday-Friday, 9am-5pm Pacific Time, to assist you with any questions or issues you may have. You can also check our status page at https://status.stacklytics.io for any updates on our service. If you have any billing disputes, please submit a ticket tagged "billing-dispute" within 14 days of the invoice date. Additionally, you can review our uptime SLA, which guarantees 99.95% uptime for Platinum tier customers, 99.9% for Diamond tier customers, and 99.5% for Gold tier customers.
