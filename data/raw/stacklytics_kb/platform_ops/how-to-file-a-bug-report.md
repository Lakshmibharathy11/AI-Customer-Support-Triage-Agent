# How to file a bug report

## Introduction to Bug Reporting
To ensure that we can assist you efficiently, it's essential to provide detailed information when filing a bug report. When encountering an issue, please check the [status page](https://status.stacklytics.io) to see if it's a known problem. If you're experiencing a common error such as a 401 invalid API key or a 429 rate limit exceeded error (error code RATE_LIMIT_EXCEEDED), you can try troubleshooting before submitting a report.

## Gathering Information
Before filing a bug report, gather as much information as possible about the issue. This includes any error codes or messages you've received, such as SDK-1042 event schema validation failure. Also, note the API endpoint you were using, which can be found in our [API documentation](https://api.stacklytics.io/v2). If you're using one of our SDKs (JavaScript, Python, or Go), please specify which one and the version you're using.

## Submitting a Bug Report
To submit a bug report, create a new ticket with a clear and descriptive title. Please include all the information you've gathered, such as error codes, API endpoints, and SDK versions. Our support team is available Monday-Friday, 9am-5pm Pacific Time, and response times vary depending on your tier: 8 hours for Gold, 4 hours for Diamond, and 1 hour for Platinum. By providing detailed information, you'll help us resolve the issue quickly and get you back to using Stacklytics.
